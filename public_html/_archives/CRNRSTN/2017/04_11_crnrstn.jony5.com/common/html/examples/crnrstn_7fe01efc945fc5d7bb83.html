The following excerpt of the user :: class object...started above...brings the login method (line 40 below) into view. Within this method are all of the necessary components for the authentication of a user with the website. One such component involves the management of the users session state.

Starting on line 48 where the parameter FORM_UN is initialized through setSessionParam(),...for the initialization of the data gathering form upon unsuccessful authentication attempt...the utilization of the CRNRSTN session_manager :: class begins to come into view.

On line 78 (scroll right), the getSessionParam() method is used to extract data from the PHP SESSION for the initialization of an encrypted cookie. The invocation of addEncryptedCookie() receives...as the parameter for the $value of the cookie...data from the users' PHP SESSION. This cookie will support the persistence of the users' PHP SESSION should said user leave the site and return within 100 days (the expire date of the cookie).

 public function login(){
  //
  // STORE AND STANDARDIZE FORM POST DATA IN STATIC PARAMS FOR PROECSSING
  self::$frm_input_username = self::$oUserEnvironment->oHTTP_MGR->extractData($_POST, 'un');
  self::$frm_input_password = md5(self::$oUserEnvironment->oHTTP_MGR->extractData($_POST, 'pwd'));
  self::$frm_input_sessionpersist = self::$oUserEnvironment->oHTTP_MGR->extractData($_POST, 'login_persist');
  
  // STORE USERNAME IN SESSION FOR FORM PREPOPULATION AFTER BAD LOGIN ATTEMPT
  self::$oUserEnvironment->oSESSION_MGR->setSessionParam('FORM_UN', self::$frm_input_username);
  
  // STANDARDIZE REQUEST FOR SESSION PERSIST
  if(strlen(self::$frm_input_sessionpersist)>5){
   self::$frm_input_sessionpersist = 1;
  }
  
  //
  // PASS UN, PWD & SESSION_PERSIST TO PRIVATE METHOD FOR WEB SERVICES REQUEST
  $this->svc_resp_ARRAY = $this->services_loginCheck(self::$frm_input_username,self::$frm_input_password,self::$frm_input_sessionpersist);
  
  //
  // CHECK FOR VALID USERNAME IN WEB SERVICES RESPONSE TO INDICATE VALID LOGIN
  if(strlen($this->svc_resp_ARRAY['USERNAME'])>0){
  
   //
   // WE HAVE VALID USER. INITIALIZE USER SESSION WITH EACH SOAP RESPONSE PARAMATER
   foreach($this->svc_resp_ARRAY as $key=>$val){
    
    //
    // THE SESSION_MANAGER :: CLASS OBJECT BELONGING TO THE CRNRSTN
    // ENVIRONMENTAL :: CLASS OBJECT (VIA USER :: CONSTRUCTOR) IS INVOKED
    // FOR SESSION INITIALIZATION
    self::$oUserEnvironment->oSESSION_MGR->setSessionParam('LOGIN_'.$key, $val);
    
    //
    // STORE ACTIVITYKEY RETURNED BY WEB SERVICE TO ENCRYPTED SESSION PERSIST
    // COOKIE (100 DAY EXPIRE) VIA CRNRSTN COOKIE_MANAGER :: CLASS OF THE
    // ENVIRONMENTAL :: CLASS OBJECT
    if($key=='SESSION_PERSIST' && $val=='1'){
     self::$oUserEnvironment->oCOOKIE_MGR->addEncryptedCookie("loginPersist", self::$oUserEnvironment->oSESSION_MGR->getSessionParam('LOGIN_ACTIVITYKEY'), time()+60*60*24*100, '/');
    }
   }
   
   //
   // RETURN SUCCESS INDICATOR FOR LOGIN
   $this->successMessage = 'Welcome back to the website '.$this->svc_resp_ARRAY['USERNAME'].'!';
   return true;
   
  }else{
   //
   // ANY FINAL THINGS TO DO FOR BAD LOGIN ATTEMPT? THROTTLING...DOS PROTECTION....ETC...
   $this->errorMessage = 'Invalid username or password provided.';
   return false;
  }
 }
