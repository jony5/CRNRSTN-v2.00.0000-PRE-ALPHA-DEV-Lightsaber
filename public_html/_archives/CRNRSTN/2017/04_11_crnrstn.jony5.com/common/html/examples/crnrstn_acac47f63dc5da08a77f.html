Here is an example of the instantiation of a cookie_manager :: class object (line 16) and the subsequent attempt to send a cookie to the client browser with an expire of 30 minutes (line 21).

The successful operation of this method...of course...requires the client browser is accepting cookies to begin with.

See documentation and sample code on the crnrstn_tmpcache :: class object for more information about the lightweight helper class for the CRNRSTN environmentals :: class object that is also shown here in this example.

Please note that if cookie encryption has been initialized through the CRNRSTN Suite ::, there are special methods that must be used to take advantage of that functionality.

require('./_crnrstn.root.inc.php');    // PATH...i.e. <?php $ROOT="../"; ? >
require($ROOT.'_crnrstn.config.inc.php');   // CRNRSTN CONFIGURATION FILE
require($ROOT.'crnrstn.tmpcache.inc.php');   // CRNRSTN TMP CACHE CLASS DEFINITION

//
// PASS CRNRSTN ENVIRONMENTALS CLASS OBJECT TO LIGTHWEIGHT TMP CACHE CONSTRUCTOR
$oTMPCACHE = new crnrstn_tmpcache($oENV);

//
// INSTANTIATE A CRNRSTN COOKIE_MANAGER :: CLASS OBJECT
if(!isset($oCOOKIE_MGR)){
 $oCOOKIE_MGR = new cookie_manager();
}

//
// SEND A COOKIE TO THE CLIENT WITH 30 MIN EXPIRE
$oCOOKIE_MGR->addCookie('chocChip[1]','value for two!', time() + 1800);

# NOTE THAT ANY COOKIE_MANAGER METHODS MUST BE INVOKED PRIOR TO OUTPUT
# BEING SENT TO THE CLIENT BROWSER...AS COOKIE DATA IS INCLUDED IN THE HTTP HEADER
