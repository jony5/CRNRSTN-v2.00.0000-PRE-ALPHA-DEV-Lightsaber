When a web page has been configured to stand on top of the CRNRSTN Suite ::, the first few lines would look something like the figure below. Line 5 of Figure 1 pulls in a script which defines a variable that is used to manage directory relativity. You see...in my sites, I put a _crnrstn.root.inc.php file in every directory where there are page(s) which are to be tied into the CRNRSTN Suite ::.

In this way, all the pages across my site will be consistent in code format (no matter how deep the page is in the web site directory structure). Each page references the _crnrstn.root.inc.php file that is in it's same directory; while within the _crnrstn.root.inc.php file, I update the $CRNRSTN_ROOT variable to reflect it's depth in the directory structure (relative to the web site root). The CRNRSTN Suite :: configuration file (stored at site root in this example) can thus be included by every page in a consistent manner...with the only variances happening within the _crnrstn.root.inc.php file to account for directory depth relative to the web site root.

On line 6 of Figure 1 below, the CRNRSTN Suite :: configuration file is included; this is where all the magic happens.
