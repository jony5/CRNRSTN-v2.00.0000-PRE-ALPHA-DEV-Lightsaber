Besides the regulation of access to an application via grantExclusiveAccess() (lines 77-81 below), IP addresses can also be managed via explicit denial (lines 89-93). In the following code example of an excerpt from the CRNRSTN Suite :: configuration file, both grantExclusiveAccess() and denyAccess() configurations are presented for demonstration purposes.

It is not recommended that both grantExclusiveAccess() and denyAccess() be used at the same time; pick the security profile that best meets your needs.
