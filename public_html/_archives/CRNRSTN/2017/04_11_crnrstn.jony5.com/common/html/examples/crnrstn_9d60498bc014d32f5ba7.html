If requiredDetectionMatches() is set to NULL, when environmental detection is fired, the user defined environmental configuration encountered having settings with the strongest correlation to your server settings (highest number of matches) will be taken. This "auto-detect" (NULL parameter) is slightly less efficient than specifying a hard number, but the detection results will be cached in the user session for all future requests, so the relative performance metrics will normalize with the other formats of method invocation after the first hit anyways.

To recap...after session initialization, there is no longer the minor performance hit of invoking this method with NULL.

Not that it needs to be said, but setting this to zero (0) provides for the fastest environmental detection as the very first $_SERVER-to-$resourceKey match will be taken as an indication of successful environmental match. So, if there is only one (1) environment for the application to run within, set this to 0...not NULL...for a slight boost to application performance.
