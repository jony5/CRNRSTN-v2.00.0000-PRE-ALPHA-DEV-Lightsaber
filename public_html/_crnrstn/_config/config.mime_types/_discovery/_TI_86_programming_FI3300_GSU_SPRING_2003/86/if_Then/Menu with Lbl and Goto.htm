If-Then Lesson 8

This lesson is similar to If-Then Lesson 7 except it will demonstrate how a Menu instruction may be used to replace several If-Then instructions when setting up a menu.

Since menus are frequently used to control the flow of programs, the TI-86 includes a separate Menu instruction.

This lesson  will demonstrate how the Menu instruction may be used with the Lbl and Goto instructions to control the flow of a program.

New Instructions Introduced:

The format of the Menu instruction is:

        :Menu ( item 1, "title 1", label 1,   item 2, "title 2", label 2,   etc... )

        where:

  • item # - an integer from 1 to 15 that identifies this item's position in the menu.
    ( a maximum of 15 menu items can be included in the Menu instruction ! )

  • "title # " - the text string that will be displayed for this item in the menu.
    ( a maximum of five characters will be displayed ! )

  • label # - the label in the program to which execution will transfer when the user selects this item.


Note: The Menu instruction is found in the [PRGM] [ CTL ] menu.

Programs are often written which contain several parts and the user is given the option of which part to run. In this example, the Lbl and Goto instructions are combined with the Menu instruction to set up a menu to allow the user to control which part of the program is executed.

When the programs runs, if the user hits the [ F1 ] key, execution will transfer to label A. If the user hits the [ F2 ] key, execution will transfer to label B, and so on.

:ClLCD
:Disp "PLEASE CHOOSE THE "
:Disp "PART YOU WANT FROM "
:Disp "THE MENU BELOW "
:Menu ( 1, "PARTA", A, 2, "PARTB", B, 3, "PARTC", C )
:
:Lbl A
:ClLCD
:Disp "THIS IS PART  A"
:Goto DONE
:
:Lbl B
:ClLCD
:Disp "THIS IS PART  B"
:Goto DONE
:
:Lbl PARTC
:ClLCD
:Disp "THIS IS PART  C"
:
:Lbl DONE
:Disp " "
:Disp "FINISHED"
  THIS IS PART  B

    FINISHED
