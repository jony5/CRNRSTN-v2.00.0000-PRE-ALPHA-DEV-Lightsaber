Functions and Graphs - Lesson 7

This lesson is similar to the previous lesson except it will show how additional options from the [ GRAPH ], [ FORMT ] menu may be included in the program to change the appearance of the graph display.

This lesson will show how the user can enter a function into a program and store that equation in the equation variable y1. The program will then use the DispG instruction to display a graph of the function.

New Instructions Introduced:

  • DispG - This instruction displays a graph of each function contained in the equation variables ( y1, y2, etc. )
  • GridOn - This instruction turns on the grid format so that grid points are displayed in rows and columns corresponding to the tick marks on each axis.
  • LabelOn - This instruction turns on the x and y axis labels and displays them on the graph.

The DispG instruction is found in the [ PRGM ], [ I / O ] menu.

The GridOn and LabelOn instructions are found in the [ GRAPH ], [ MORE ], [ FORMT ] menu. ( or you can get them from the "catalog" ! )

Note: The function must be entered in terms of  x. It is suggested that you use the [ x-VAR ] key to enter  x  in the function.

This example will demonstrate how the user can enter a function into a program and store the function in the equation variable y1. First, the function must be entered as a string and stored in the string variable Y using the InpSt instruction. Then, the St>Eq instruction is used to convert the string variable Y into a function and store it in the equation variable y1.

The program will then pause and allow the user to hit the [ ENTER ] key to view the graph. After the program ends, the graph will remain on the screen and the graph menu will be made available for the user to choose options to explore the graph.

In addition, this program will turn on the grid points and label for each axis on the graph.

Note: I suggest that you call the SETUP86 program to reset the calculator settings before running any program that involves graphing to insure that the program runs the way you are expecting it to. If you don't do this, you may be surprised by some unexpected results!

Tip: If you place a Pause instruction immediately following the DispG instruction, the program will pause until the user hits the [ ENTER ] key. This will allow time for the user to examine the graph before the bottom of the graph is covered up by the graph menu!

:SETUP86
:ClLCD
:Disp "ENTER THE FUNCTION"
:Disp "( USE x-VAR KEY FOR x )"
:InpSt "Y(x) =   ", Y
:St>Eq ( Y , y1 )
:Outpt( 7 , 1 , "HIT ENTER TO ")
:Outpt( 8 , 1 , "VIEW GRAPH " )
:Pause
:ClLCD
:GridOn
:LabelOn
:DispG
:Pause
     ENTER THE FUNCTION
    ( USE x-VAR KEY FOR x )
    Y(x) = x^2  -  9


    HIT  ENTER TO
    VIEW  GRAPH
