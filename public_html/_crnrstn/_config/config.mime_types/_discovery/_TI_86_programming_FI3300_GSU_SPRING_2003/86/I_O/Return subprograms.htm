I/O Lesson 18

This lesson will demonstrate how one program can call another program. This technique is similar to using subroutines in other programming languages. On the TI-86, any stored program can be called from another program as a subroutine. To call one program from another, simply enter the subprogram name at any point in the calling (main) program.

When the program name is encountered as the calling program runs, the next command executed is the first command in the subroutine. Execution returns to the next line in the calling program when the subroutine either ends or hits a Return instruction.

New Instructions Introduced:

  • Return - This is an optional instruction that may be placed at the end of a subroutine which causes execution to return to the main program. If the Return statement is left out, an implied Return is assumed, and execution will still return to the calling program. This feature allows any program stored in the TI-86 to be called as a subroutine to any other program, even if a Return instruction is not included!


The Return instruction is located in the [PRGM][CTL] menu.

Note: In this example, the Return instruction is optional. The subprogram will return to the main program even if the Return statement is omitted!

PROGRAM: MAINPROG
:ClLCD
:Disp "This is the main "
:Disp "program"
:Outpt( 7 , 1 , "HIT ENTER
TO CONTINUE")
:Pause
:SUBPROG
:ClLCD
:Disp "You are now back in"
:Disp "the main program"

PROGRAM: SUBPROG
:ClLCD
:Disp "The main program has"
:Disp "called the subprogram"
:Disp " "
:Disp "This is the"
:Disp "subprogram"
:Outpt( 7 , 1 , "HIT ENTER
TO CONTINUE")
:Pause
:Return
